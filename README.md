# project-dtmfSignalSos-androidApp

### Проведено исследование работы механизма передачи в аудиовыход каких либо данных:

Причина, по которой мы **не можем** проигрывать другой звук через аудиоканал во время звонка, заключается в том, что система Android обычно нет прямой связи фактического потока данных GSM с какой-либо частью программного стека системы, который можно использовать для его изменения; 

То,  есть получается что он зеркально копирует фактические данные телефонного звонка (микрофон / динамик) в систему + есть небольшой контроль, например, отключения микрофона, отправку DTMF (тонов) и отмена вызова.

Точная маршрутизация звука происходит в телефонном чипе, поэтому контроль над ней невозможен.

Если брать возможность обхода этого механизма через Bluetooth (Эмулировать гарнитуру), то это снова **не получится**, так как аудиопоток Bluetooth декодируются и зеркалируются в микросхеме (DAIBT - технология для прямого аудиоинтерфейса), чтобы было как можно меньше задержек и энергоэффективно.

---

> Есть производители, которые реализуют схему аудиовызова по другому (некоторые чипы Qualcomm). Они предоставляют специфичный способ вставки звука в канал связи GSM-вызова (MSM8960, APQ8064) [https://stackoverflow.com/a/15205455/438039](https://stackoverflow.com/a/15205455/438039) - вот на них можно вставить по идеи свой звук в GSM вызов.

Возможно заранее записать последовательность DTMF сигналов, в которую можно зашить информацию

Данный проект написан в рамках хакатона - https://demhack.ru

Приложение представляет собой систему для посылки сигнала SOS.
